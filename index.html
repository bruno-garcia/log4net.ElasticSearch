High level feature list

.NET 4+ & Mono Support

log4net.Elasticsearch supports .NET 4, 4.5.x, and Mono/Xamarin out of the box.

Asynchronous

Fully asynchronous to avoid application slow downs. Buffer size is fully tunable and lossy option will ensure resource consumption constraints.

Elasticsearch Conventions

Built with Elasticsearch conventions in mind including camelCase types and properties.

Rolling Indices

Log to one index or create a new index each day and expire as needed.

Open Source

Licensed under MIT. Contributions are more than welcome!

Nuget

Latest packages are always available on Nuget. Zero dependencies (besides log4net, of course).

1. Install from Nuget

In Visual Studio, open the Package Manager console and issue the install command:

Install-Package log4net.ElasticSearch
OR
Install-Package log4net.ElasticSearch -InstallPrerelease (will install latest pre-release version)

1. Configure log4net

Move the plugin JS file to your server and link to it before the closing body tag.

Please note that you must include jQuery (v1.7 or later) before the slider JS file.

<script src="jquery-1.7.1.min.js"></script>
<script src="jquery.yourplugin.min.js"></script>

The Markup

Add the following markup

<ul class="thumbnails">
<li class="span4"> <a href="#" class="thumbnail"> <img src="http://placehold.it/360x268" alt=""> </a> </li>
....
</ul>

Call Plugin

Just before the closing body tag (after any JS file includes), call the plugin with code below:

<script>
$(document).ready(function () {
	$('.selector').yourPlugin();
});
</script>
